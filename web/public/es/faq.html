¿Qué es Cocxy Terminal?

Cocxy Terminal es una terminal nativa para macOS construida para desarrolladores que trabajan con agentes de código. Detecta estado de agentes en tiempo real, incluye revisión de código integrada, workspace Markdown, renderizado GPU con Metal, sesiones remotas persistentes y cero telemetría.

¿Cocxy Terminal es gratis?

Sí. Cocxy es gratuito bajo licencia MIT. No hay suscripción, cuenta obligatoria ni nivel de pago. El código está publicado en GitHub.

¿Qué agentes de código soporta?

Detecta agentes de código compatibles mediante hooks, secuencias OSC, patrones de salida y heurísticas de tiempo. Puedes agregar agentes personalizados en ~/.config/cocxy/agents.toml.

¿Cocxy recopila datos?

No. Cocxy no incluye analíticas, tracking ni reporte automático de crashes. La revisión de actualizaciones consulta el appcast firmado. GitHub, SSH, providers remotos, DB/Cloud y otras herramientas externas solo se usan cuando tú las configuras o ejecutas.

¿Cuáles son los requisitos?

macOS 14.0 Sonoma o posterior, Apple Silicon o Intel, y cerca de 60 MB de espacio instalado. Una conexión a internet solo es necesaria para descargar versiones, usar Homebrew o invocar servicios externos configurados por ti.

¿Cómo instalo Cocxy en macOS?

Con Homebrew: brew tap salp2403/tap && brew install --cask cocxy. También puedes descargar el DMG firmado y notarizado desde GitHub Releases y arrastrar la app a /Applications.

¿Cómo actualizo Cocxy?

Si usas Homebrew, ejecuta brew update && brew upgrade --cask cocxy. La app también usa Sparkle con firma EdDSA para revisar actualizaciones del appcast firmado.

¿Qué es el panel de revisión de código?

Es un panel nativo que muestra archivos modificados por una sesión de agente. Permite navegar diffs, comentar líneas, aplicar sugerencias y enviar feedback al agente desde el flujo local. Se abre con Cmd+Option+R.

¿Puedo usar Cocxy como terminal diaria?

Sí. Incluye tabs, splits, búsqueda en scrollback, imágenes inline, ligaduras, integración zsh/bash/fish, SSH, AppleScript, atajos configurables, temas y CLI local.

¿Funciona con mi configuración zsh o bash?

Sí. La integración de shell está diseñada para coexistir con frameworks y prompts existentes. Cocxy restaura variables como ZDOTDIR, HOME y XDG_CONFIG_HOME antes de cargar tu entorno.

¿Cocxy puede conectarse a servidores remotos?

Sí. Remote Workspaces usa OpenSSH ControlMaster, perfiles locales, Keychain opcional, túneles, SFTP y tmux. No requiere instalar un backend propietario en el servidor.

¿Qué es el workspace Markdown?

Es un panel Markdown nativo junto a la terminal. Permite editar, previsualizar Mermaid y KaTeX, usar callouts, footnotes, resaltado de sintaxis, QuickLook offline y exportación desde flujos locales.

¿Soporta temas personalizados?

Sí. Puedes usar los temas incluidos o colocar un archivo .toml en ~/.config/cocxy/themes/. La app también puede alternar tema claro/oscuro según la apariencia de macOS.

¿Dónde vive la configuración?

La configuración principal vive en ~/.config/cocxy/config.toml. Los agentes se configuran en agents.toml. También puedes usar .cocxy.toml dentro de un proyecto para overrides locales.

¿Hay CLI?

Sí. El comando cocxy controla tabs, splits, ventanas, sesiones, agentes, hooks, revisión, Markdown, navegador, remoto, proxy, plugins, temas, timeline y diagnósticos. Usa cocxy help para ver la referencia.

¿Soporta plugins?

Sí. Los plugins responden a eventos de terminal como inicio/fin de sesión, agente detectado, cambio de estado, comando completado, tab creado/cerrado y cambio de directorio. Se ejecutan con sandbox, timeout y permisos explícitos.

¿Cómo reporto un bug?

Abre un issue en GitHub Issues con versión de Cocxy, versión de macOS, chip y pasos para reproducir. Para vulnerabilidades, escribe a security@cocxy.dev.

¿Cómo desinstalo Cocxy?

Si instalaste con Homebrew: brew uninstall --cask cocxy. Si instalaste con DMG, mueve la app a Trash. Para borrar configuración y sesiones locales elimina ~/.config/cocxy/ y ~/Library/Application Support/dev.cocxy.terminal/.
