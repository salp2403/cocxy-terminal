Funciones principales

Cocxy Terminal está diseñada como una app nativa para macOS: terminal diaria, paneles de desarrollo, automatización local y flujos con agentes sin backend propietario.

Detección local de agentes

Cocxy detecta sesiones de agentes de código mediante hooks locales, secuencias OSC, patrones de salida y heurísticas de tiempo. Las señales se procesan por superficie y alimentan dashboard, timeline, sidebar, Smart Routing y rutas de feedback.

Ejecuta cocxy setup-hooks para configurar agentes compatibles instalados localmente. Los perfiles personalizados se auditan en ~/.config/cocxy/agents.toml.

Panel de revisión de código

Muestra archivos modificados, diffs, comentarios inline, acciones por hunk, estadísticas, templates de respuesta y feedback estructurado hacia el agente. El objetivo es revisar cambios reales sin abandonar la terminal.

El panel entrega feedback por la PTY original, aplica suggestions localmente con guards contra rutas inseguras y puede preparar commits o pull requests desde el flujo Git local.

Panel GitHub integrado

El panel GitHub lista pull requests, issues, checks y review threads del repo activo usando tu propia sesión de gh. Cocxy no almacena tokens propios.

La CLI incluye comandos como cocxy github status, cocxy github prs y cocxy github refresh para automatizar el panel desde scripts locales.

Workspace Markdown

Editor de fuente, preview local, modo split, Mermaid, KaTeX, callouts, footnotes, tabla de contenido, resaltado de sintaxis, búsqueda, adjuntos de imagen y exportación HTML/PDF. El editor y preview usan colores legibles en temas claros y oscuros.

QuickLook offline

Los notebooks de Cocxy generan previews QuickLook sin red. El contenido necesario se conserva localmente, y el bundle verifica que la extensión y metadatos se empaqueten junto a la app.

Motor GPU con CocxyCore

CocxyCore es el motor de terminal escrito en Zig con ABI C estable y renderizado Metal. Mantiene parser, scrollback, hyperlinks, bloques semánticos, protocolo de imágenes y diagnósticos expuestos al bundle Swift.

Workspaces remotos

Perfiles SSH, ControlMaster, jump hosts, túneles, SFTP, tmux, proxy, relay y daemon opcional para sostener sesiones remotas persistentes sin servidor Cocxy obligatorio. Las llaves se importan y gestionan localmente con Keychain cuando el usuario lo decide.

Navegador integrado

Preview de servidores locales, documentación y apps web junto a la terminal. Incluye perfiles aislados, bookmarks, descargas, find, DevTools, panel overlay o split, e integración con chips de puertos detectados.

Web Terminal local

Expone una terminal local por HTTP cuando lo activas con cocxy web-start --port 8080. Sirve para pair programming, asistencia interna o dashboards privados, con control explícito de inicio, estado y cierre.

Sistema de plugins

Plugins descentralizados con sandbox, permisos explícitos, timeout, fuentes locales y eventos de terminal. Pueden vivir en usuario, proyecto o bundle, y se ejecutan con el contexto mínimo que el evento necesita.

Configuración por proyecto

Un .cocxy.toml en el repo permite ajustar fuente, opacidad, patrones extra, keybindings y preferencias de workspace solo para ese proyecto, con hot reload conservador.

AppleScript y Shortcuts

Automatiza tabs, comandos, splits, foco y conteo de sesiones desde herramientas de macOS. Las acciones de Shortcuts.app ejecutan solo operaciones locales y describen claramente su frontera de privacidad.

Integración de shell

Marcadores OSC para directorio, prompt, límites de comando, bloques, hyperlinks y estado semántico. La integración conserva configuraciones zsh, bash y fish existentes y degrada de forma segura cuando un shell no soporta una señal.

Cero telemetría

Sin analytics, tracking, reporte automático de crashes ni backend propietario. La red solo aparece cuando instalas, actualizas o usas una herramienta externa configurada por ti; los audits del repo bloquean SDKs de telemetría y copy público impreciso.

CLI local

La CLI cocxy controla tabs, ventanas, splits, sesiones, agentes, hooks, Markdown, remoto, plugins, GitHub, web terminal, temas y diagnósticos mediante un Unix Domain Socket local con autenticación por usuario.
