Guía de uso

Documentación de Cocxy Terminal

Cocxy Terminal es una terminal nativa para macOS construida para flujos con agentes de código. Mantiene el trabajo local, usa renderizado acelerado por GPU, ofrece revisión de código integrada, workspace Markdown, sesiones remotas y cero telemetría.

Instalar Homebrew, DMG firmado, primera apertura y actualizaciones. Flujos Agentes, revisión de código, Markdown, remoto y CLI. Privacidad Cero telemetría, configuración local y red explícita.
Requisitos del sistema

macOS 14.0 Sonoma o posterior, Apple Silicon o Intel, y cerca de 60 MB de espacio en disco. El acceso a internet solo es necesario para descargar actualizaciones, instalar desde Homebrew o usar herramientas externas que tú ejecutes.

Instalar con Homebrew

La ruta recomendada es brew tap salp2403/tap && brew install --cask cocxy. Para actualizar después, ejecuta brew update && brew upgrade --cask cocxy. El cask instala la app y deja disponible el binario cocxy incluido en el bundle.

Instalar con DMG

Descarga el DMG más reciente desde GitHub Releases, abre la imagen y arrastra Cocxy Terminal a /Applications. La app está firmada y notarizada por Apple; si macOS muestra un aviso inicial, usa clic derecho y Open.

Primera apertura

Al iniciar, Cocxy crea ~/.config/cocxy/ con valores seguros por defecto. Puedes reabrir la guía inicial desde el menú Help y cambiar idioma, shell, apariencia, atajos, plugins y paneles desde Preferences.

Privacidad y red

Cocxy no incluye analíticas, tracking ni reporte automático de crashes. La revisión de actualizaciones usa el appcast firmado de Cocxy. Cualquier provider remoto, GitHub CLI, SSH, nube o base de datos se ejecuta solo cuando tú lo configuras y lo invocas de forma explícita.

Agentes de código

Cocxy detecta agentes de código compatibles mediante hooks, secuencias OSC, patrones y heurísticas de tiempo. Puedes agregar agentes personalizados en ~/.config/cocxy/agents.toml y ejecutar cocxy setup-hooks para configurar integraciones locales compatibles.

Revisión de código integrada

El panel de revisión muestra archivos tocados por una sesión de agente, diffs, comentarios inline y acciones para aceptar o revertir cambios de forma granular. Ábrelo con Cmd+Option+R o desde la paleta de comandos.

Workspace Markdown

El panel Markdown permite editar y previsualizar documentos junto a la terminal. Incluye Mermaid, KaTeX, callouts, footnotes, resaltado de sintaxis, tabla de contenido, QuickLook offline y exportación a HTML/PDF desde flujos locales.

Espacios remotos y DB/Cloud

Remote Workspaces usa OpenSSH, perfiles locales, Keychain opcional, túneles, SFTP y tmux. Los helpers DB/Cloud son plugins locales que muestran comandos y previews redactados; no guardan credenciales nuevas ni ejecutan llamadas automáticas.

CLI local

El binario cocxy controla tabs, splits, sesiones, paneles, plugins, temas, timeline y diagnósticos mediante un Unix Domain Socket local con permisos de usuario. Prueba cocxy status, cocxy new-tab, cocxy list-tabs y cocxy help.

Migrar desde versiones v0.x

Antes de actualizar, revisa Releases y mantén una copia de ~/.config/cocxy/ si usas perfiles SSH, snippets, temas o plugins personalizados. Las versiones nuevas migran archivos locales de forma conservadora y preservan configuración existente siempre que el TOML sea válido.

Solución rápida de problemas

Si una preferencia no cambia, guarda config.toml y revisa que no tenga errores TOML. Si Homebrew no ve una versión nueva, ejecuta brew update. Para resetear, cierra Cocxy y elimina ~/.config/cocxy/; esto también borra perfiles, bookmarks y datos locales guardados ahí.
